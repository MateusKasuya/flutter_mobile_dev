# Problema — API de login retorna 202, não 200

[[Home]]

---

## Descrição

O app retornava "Erro ao fazer login" mesmo com credenciais corretas.
Comportamento esperado: navegar para HomeScreen com o token.

## Contexto

`auth_service.dart`, função `login()`. Reproduzível sempre que o CPF e senha estavam corretos.

## Investigação

Testado o endpoint diretamente com `curl`:

```bash
curl -s -w "\nHTTP_STATUS:%{http_code}" -X POST \
  "http://fretefacilweb.ccmcloud.com.br:8624/sftlogin/login" \
  -H "Content-Type: application/json" \
  -d '{"cpfusuario": "07069953925", "senhausuario": "1"}'
```

Resultado: a API retornou **HTTP 202** (Accepted) com o token no body.
O código só tratava **200** como sucesso — o 202 caía no bloco de erro.

## Solução

```dart
// antes
if (response.statusCode == 200) {

// depois
if (response.statusCode == 202) {
```

## Root cause

Assumimos que a API seguia a convenção REST de retornar 200 para login bem-sucedido, mas ela usa 202 (Accepted). Sempre validar o status real da API antes de codificar.
