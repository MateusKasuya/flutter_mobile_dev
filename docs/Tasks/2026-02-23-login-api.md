# Task — Login com autenticação via API

[[Tasks/_index|Tasks]]

---

## Contexto

O app estava com apenas um "Hello World". Para começar o fluxo de autenticação da frota, precisávamos de uma tela de login que se comunicasse com a API real e retornasse um token JWT.

## Objetivo

Usuário preenche CPF e senha, o app faz POST na API de login, recebe o token e navega para a tela inicial.

---

## Branch

Esta task foi executada diretamente na `main` por ser a primeira feature do projeto.
A partir da próxima task, o fluxo de branches estará ativo.

## Implementação

### Passo 1 — Adicionar dependência HTTP

Adicionar `http: ^1.2.0` em `pubspec.yaml` e rodar `flutter pub get`.

### Passo 2 — Criar serviço de autenticação

Criar `lib/services/auth_service.dart` com função `login(cpfusuario, senhausuario)`:
- Constrói a URL com `Uri.http('fretefacilweb.ccmcloud.com.br:8624', '/sftlogin/login')`
- Faz POST com body JSON `{ cpfusuario, senhausuario }`
- Trata status **202** como sucesso e retorna `data['access_token']`
- Lança `Exception` com `data['message']` em caso de erro

### Passo 3 — Criar tela de login

Criar `lib/screens/login_screen.dart` como `StatefulWidget`:
- Recebe `loginFn` como parâmetro opcional (padrão: função `login` real)
- Campos: CPF (teclado numérico) e Senha (obscureText)
- Botão "Entrar" — desabilitado durante loading
- Exibe `CircularProgressIndicator` enquanto aguarda resposta
- Em sucesso: navega para `HomeScreen` com o token
- Em erro: exibe mensagem em vermelho

### Passo 4 — Criar tela pós-login

Criar `lib/screens/home_screen.dart` recebendo o token e exibindo na tela.

### Passo 5 — Atualizar main.dart

Trocar o `Scaffold` com "Hello World" por `LoginScreen()`.

### Passo 6 — Widget tests

Criar `test/screens/login_screen_test.dart` com 3 casos:
- Renderiza campos CPF, Senha e botão Entrar
- Login com sucesso → navega para HomeScreen com o token
- Login com falha → exibe mensagem de erro

## Problemas encontrados

- API retorna HTTP **202**, não 200 → corrigido após debug com `curl`
- CORS bloqueia requisição no Chrome → não é bug do app; testar em iOS Simulator ou Android
